Shardingsphere分表注意事项

# 涉及需要分表的count操作

代码内，执行的sql：

select count(distinct o.user_id) buyUserCount from orders o left join user_child uc on o.user_id = uc.user_id left join user_addr ua on o.user_id = ua.user_id where o.status in (?, ?, ?, ?) and o.pay_time>=? and o.pay_time<=?
1

实际执行（当查询不涉及分表字段，定位不出实际需要查哪张表，shardingsphere会查询所有分表）：

select DISTINCT o.user_id buyUserCount from orders2019 o left join user_child uc on o.user_id = uc.user_id left join user_addr ua on o.user_id = ua.user_id where o.status in (?, ?, ?, ?) and o.pay_time>=? and o.pay_time<=?

select DISTINCT o.user_id buyUserCount from orders2019 o left join user_child uc on o.user_id = uc.user_id left join user_addr ua on o.user_id = ua.user_id where o.status in (?, ?, ?, ?) and o.pay_time>=? and o.pay_time<=?
1
2
3

以上查询结果为多个user_id的集合，shardingsphere回合并这些集合，返回String类型的数量，当数量为0的时候返回null

鉴于以上分析，使用count查询时，无法使用((Number) sqlQuery.uniqueResult()).intValue()返回int数量。需要改为： Object retNum = sqlQuery.uniqueResult(); return retNum == null ? 0 : Integer.parseInt(retNum.toString());

# resultSet与model的映射

model类型需要和数据库字段类型对应，否则会出现异常，如：当数据库为varchar类型，model为int类型的时候，分表查询时会出现类型转换异常，但是当涉及分表字段，定位出准确表名的时候又不会报错

# 使用group by函数时，会被自动加上order by

order by的内容与group by 后的内容相同

# 使用逻辑表名来明确列所属表，且sql内包含group by语句

不使用逻辑表名来明确列所属表，以下sql为例，orders为逻辑表名（sql内使用的的表名），实际查询的表为orders2019和orders2020

select orders.activity_id activityId,act_item.act_item_id actItemId, act_item.title actTitle, count(1) orderNums from orders,act_item where act_item.act_item_id = orders.act_item_id and orders.status in(?, ?, ?, ?) and act_item.status =? AND act_item.expires_at>? group by orders.activity_id,act_item.act_item_id
1

上述sql使用逻辑表名明确列的归属，经过shardingsphere的处理后变成以下两个sql，由于被加上order by，且orders没被替换为实际的表名，导致sql报错

rders2019.activity_id activityId,act_item.act_item_id actItemId, act_item.title actTitle, count(1) orderNums from orders2019,act_item where act_item.act_item_id = orders2019.act_item_id and orders2019.status in(?, ?, ?, ?) and act_item.status =? AND act_item.expires_at>? group by orders2019.activity_id,act_item.act_item_id ORDER BY orders.activity_id ASC,act_item.act_item_id ASC

select orders2020.activity_id activityId,act_item.act_item_id actItemId, act_item.title actTitle, count(1) orderNums from orders2020,act_item where act_item.act_item_id = orders2020.act_item_id and orders2020.status in(?, ?, ?, ?) and act_item.status =? AND act_item.expires_at>? group by orders2020.activity_id,act_item.act_item_id ORDER BY orders.activity_id ASC,act_item.act_item_id ASC
1
2
3

以上错误只需要给逻辑表名添加上别名即可，可以是符合语法的任何别名

select orders.activity_id activityId,act_item.act_item_id actItemId, act_item.title actTitle, count(1) orderNums from orders orders,act_item where act_item.act_item_id = orders.act_item_id and orders.status in(?, ?, ?, ?) and act_item.status =? AND act_item.expires_at>? group by orders.activity_id,act_item.act_item_id
1

# 数据库的表字段名不能为channel

疑似shardingsphere的关键词，带有channel字段名的sql语句不能被正确解析。
