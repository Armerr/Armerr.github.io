服务器安装clash

# 一、下载安装包

1.通过sftp上传安装包 下载地址：https://github.com/Dreamacro/clash 选择符合系统最新的release版本上传到服务器 2.命令下载

# 新建文件夹
mkdir /opt/clash
cd /opt/clash
# 下载包
wget https://github.com/Dreamacro/clash/releases/download/v1.16.0/clash-linux-amd64-v1.16.0.gz
# 解压
gzip -dv clash-linux-amd64-v1.16.0.gz
# 重命名为clash
mv clash-linux-amd64-v1.16.0 clash
# 执行权限
chmod u+x clash

# 四、设置系统代理(设置全部流量走代理软件，如不需要全局代理可不设置)

sudo vim /etc/environment
# 加入以下三行数据
export http_proxy="http://127.0.0.1:7890"
export https_proxy="http://127.0.0.1:7890"
export no_proxy="localhost, 127.0.0.1"
# 修改sudo文件
sudo vim sudo
# 文件尾部加如下一行
Defaults env_keep+="http_proxy https_proxy no_proxy"
# 重启服务器
reboot

# 五、控制面板

在如下地址中填入clash的ip+端口，如未设置密码则留空,服务器需要将clash的端口暴露出来 http://clash.razord.top/#/proxies

# 六、设置开机自启动

# 将配置文件转移到/etc中
sudo mv ~/.config/clash /etc
# 添加启动信息
sudo vim /etc/systemd/system/clash.service
# 输入以下内容，clash -d的意思是指定配置文件路径，这里已经改成了/etc/clash
[Unit]
Description=clash daemon

[Service]
Type=simple
User=root
ExecStart=/opt/clash/clash -d /etc/clash/
Restart=on-failure

[Install]
WantedBy=multi-user.target

# 重新加载systemctl daemon
sudo systemctl daemon-reload

# 启动Clash
sudo systemctl start clash.service

# 设置Clash开机自启动
sudo systemctl enable clash.service

# 重启Clash命令
sudo systemctl restart clash.service

# 查看Clash运行状态
sudo systemctl status clash.service
